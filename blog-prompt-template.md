# Blog Generation Request Template

Use this template to generate competitive analysis blog posts. Fill in all sections below with your research data, then ask Cursor to generate the blog post.

## ⚠️ CRITICAL REQUIREMENT

**Every quote, claim, and piece of evidence MUST have an exact source URL.**

When filling out this template:
- Include the full URL for every G2 review page
- Provide exact Reddit URLs for every quote
- Provide URLs for all pricing pages, documentation, and resources
- No quote should be included without a verifiable source URL

## Instructions for Use

### Step 1: Fill Out This Template

1. Copy this template to a new file
2. Fill in ALL sections with your research data
3. Be thorough - the more data you provide, the better the blog post
4. Include direct quotes with proper attribution
5. Gather all relevant links upfront

### Step 2: Generate the Blog Post

Once your template is complete, use this prompt in Cursor:

```
Generate a competitive analysis blog post using the blog-guide.md style guide and the research data I've provided in blog-prompt-template.md. 

Note: This data was collected using Parallel MCP for efficient web search.

Follow all formatting, SEO, and content guidelines from blog-guide.md. The blog post should be saved to src/data/blog/ with the filename format: company-a-vs-company-b-vs-company-c-vs-company-d.md

Ensure:
- Conversational, natural opening
- Question-based H2 sections
- All evidence properly cited with exact source URLs
- Every quote has a clickable source link
- Comparison tables included
- Sources & links section at the end
- Excerpt is exactly 150-160 characters
```

### Step 3: Review Generated Blog

After Cursor generates the blog, review it against the Quality Checklist in `blog-guide.md`:

**Essential Checks:**
- [ ] Frontmatter is complete and properly formatted
- [ ] Excerpt is 150-160 characters
- [ ] Opening is conversational and SEO-optimized
- [ ] All H2 sections are questions (where appropriate)
- [ ] Comparison tables render correctly
- [ ] All links are clickable
- [ ] Every quote has a source URL
- [ ] All evidence is properly cited with URLs
- [ ] Sources & Links section is complete
- [ ] File is saved in correct location with correct name

### Step 4: Publish

Once reviewed and approved, the blog is ready for publication!

## Tips for Gathering Data

### Using Parallel MCP for Efficient Research

**Parallel MCP** enables fast, comprehensive web search across multiple sources. Use it to:
- Search for multiple companies simultaneously
- Gather G2 reviews, Reddit discussions, and pricing data in parallel
- Access comprehensive competitive intelligence efficiently

**Important:** Always wait for all Parallel MCP tasks to complete before proceeding with blog generation.

### G2 Reviews
1. Use Parallel MCP to search for company G2 pages
2. Visit company's G2 page and copy the URL
3. Sort reviews by "Most Recent"
4. Look for patterns in pros/cons
5. Note specific mention counts from G2's analysis
6. **Save direct quotes with the G2 page URL** (or specific review URL if available)

### Reddit Research
1. Use Parallel MCP to search for company names in relevant subreddits
2. Look for comparison discussions
3. Note user sentiment (positive/negative)
4. Capture direct quotes with context
5. **Ensure exact Reddit URLs are captured for every quote**

### Pricing Data
1. Use Parallel MCP to find official pricing pages quickly
2. Visit official pricing pages
3. Document all tiers and costs
4. Look for hidden fees or add-ons
5. Note what's included/excluded
6. Calculate true cost if different from advertised

### Competitive Intelligence
1. Use Parallel MCP to search for company information
2. Visit company websites
3. Read "About" and positioning pages
4. Check recent news/funding announcements
5. Review case studies if available
6. Note target audience and messaging

**💡 Pro Tip:** Parallel MCP can search multiple companies at once, significantly reducing research time from hours to minutes.
